MySQL 的基本架构

基本架构

  1. ⼤体来说，MySQL 可以分为 Server 层和存储引擎两部分。
    Server 层包括：连接器、查询缓存、分析器、优化器、执⾏器等，涵盖了 MySQL 的⼤多数核⼼服务功能，以及所
  2. 有的内置函数（如：⽇期、时间、数学和加密函数等），所有跨存储引擎的功能都在这⼀层实现，⽐如：存储过
    程、触发器、视图等等。
  3. 存储引擎层负责：数据的存储和提取。其架构是插件式的，⽀持 InnoDB、MyISAM 等多个存储引擎。从
    MySQL5.5.5 版本开始默认的是InnoDB，但是在建表时可以通过 engine = MyISAM 来指定存储引擎。不同存储引
    擎的表数据存取⽅式不同，⽀持的功能也不同。
  4. 不同的存储引擎共⽤⼀个 Server 层，也就是从连接器到执⾏器的部分
